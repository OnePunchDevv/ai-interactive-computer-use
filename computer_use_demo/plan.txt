Phase 3 — Session Manager & Agent Runner (1 hour)
Step 3.1 — agent_runner.py:

Wraps loop.py's sampling_loop()
Accepts a session ID and initial message
Writes every message/tool event to DB as it streams
Publishes each event to a per-session asyncio.Queue
Manages its own display_num and DISPLAY env var per session
Step 3.2 — session_manager.py:

Maintains Dict[session_id, asyncio.Task] — no fixed cap
create_session() — picks next free display number (:2, :3, ...), creates DB record, starts Xvfb on that display, assigns a VNC port
run_agent(session_id, message) — spawns new asyncio.Task per request (non-blocking), each task is independent
subscribe(session_id) — returns the queue for SSE streaming
stop_session(session_id) — cancels task, kills display process, updates DB
Uses asyncio.Lock per session for message ordering (prevents race conditions)
Dynamic display/port allocation using a asyncio.Semaphore-guarded counter
Key concurrency design: Every session runs in its own asyncio.Task with its own DISPLAY=:N env var pointing to a dedicated Xvfb process. No shared display state. Sessions are fully isolated.

Phase 4 — FastAPI Routers (45 min)
Step 4.1 — routers/sessions.py:

POST /sessions — create new session, return {id, title, status, vnc_url}
GET /sessions — list all sessions with metadata
GET /sessions/{id} — get single session
DELETE /sessions/{id} — stop and delete session
Step 4.2 — routers/messages.py:

POST /sessions/{id}/messages — send a user message, triggers agent (non-blocking, returns 202 Accepted)
GET /sessions/{id}/messages — return full message history from DB
GET /sessions/{id}/stream — SSE endpoint: streams real-time events from the session queue (text chunks, tool calls, tool results, status changes)
Step 4.3 — routers/vnc.py:

GET /sessions/{id}/vnc-info — returns the noVNC URL and port for that session's display
Step 4.4 — main.py: CORS, lifespan events (startup DB init, shutdown cleanup), router registration

Phase 6 — Frontend (30 min)
A clean, minimal HTML/JS app (no React, no Swagger):

Layout:

Left sidebar: Session list + "New Session" button
Center: noVNC iframe (connects to the selected session's VNC port)
Right panel top: Chat messages (user, assistant, tool calls rendered distinctly)
Right panel bottom: File/status area
JS behavior:

On "New Session": POST /sessions, add to sidebar
On session click: load history, open VNC iframe, start SSE stream
SSE stream listener: append messages in real-time as they arrive
Tool calls shown with a distinct style (e.g., gray code block with tool name)
Spinner while agent is running, input enabled only when idle
After task completes, re-enable input for new task in same session
